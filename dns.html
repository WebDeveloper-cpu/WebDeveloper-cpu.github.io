How DNS Translates URLs to IP Addresses

The Anatomy of a URL

A URL is a map. To understand where you are going, you must break the address down into its functional parts:

  • Protocol: https:// (The language used to communicate)
  • Subdomain: blog. (A specific section of the site)
  • Domain Name: example.com (The human-friendly name)
  • Path: /contact (The specific resource/file)

The 4 Steps of DNS Resolution

When you press "Enter," your browser initiates a four-part conversation to translate that URL into a machine-readable IP Address.

1. DNS Recursor

Acts as your personal librarian. It takes the browser's request and handles the heavy lifting of querying the other servers until it finds the correct IP.

2. Root Nameserver

The first point of contact in the DNS hierarchy. It doesn't know the specific IP, but it knows which server manages the .com or .org registry.

3. TLD Nameserver

Manages the Top-Level Domain (TLD). It holds the info for every registered domain with that extension and points the search toward the domain's own server.

4. Authoritative Nameserver

The "Source of Truth." This server is maintained by the website owner and holds the actual A Record containing the IP address.

Summary of the Hierarchy

DNS is a tree-like structure, moving from general to specific:

  1. Root Level: The start of the internet (.)
  2. TLD Level: The extension (.com, .net)
  3. Second-Level Domain: The brand (google, example)
  4. Subdomain: The specific server (docs, mail)
