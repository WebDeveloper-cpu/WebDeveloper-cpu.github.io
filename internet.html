The Internet & Networking Basics

The internet is so deeply integrated into our daily lives that we rarely think about how it works. At its most basic level, the internet is a massive, global network of computers connected together to share information.

To understand how this global network functions, it helps to break it down into core fundamental concepts: local networks (LANs), global networks (WANs), how computers find each other (IP addresses), and how they communicate (protocols).

1. What is a Network? (LAN vs. WAN)

A network is simply two or more devices connected together so they can communicate.

  • LAN (Local Area Network): This is a network that covers a small, localized area, like your home, school, or office. When your phone, laptop, and smart TV are all connected to your home Wi-Fi router, they form a LAN. They can easily share files or communicate with each other locally without even needing the internet.
  • WAN (Wide Area Network): When you connect multiple LANs across large geographical distances (cities, countries, or continents), you get a WAN.

The Internet is the ultimate WAN. It is a global "network of networks" connecting billions of local LANs together using fiber-optic cables running underground and across ocean floors.

2. How Devices Connect: Clients, Servers, and Routers

To navigate the internet, devices play specific roles:

  • Clients: These are the devices that request information. Your smartphone, laptop, or smart TV acting on your command is a client. When you open a web browser and search for a website, your device is making a request.
  • Servers: These are powerful, specialized computers that serve or store information (websites, videos, database files). They are kept powered on 24/7. When you visit a website, a server somewhere in the world sends that website's files to your client device.
  • Routers: Think of a router as a digital traffic cop. When you send data from your device, the router looks at where it's supposed to go and directs it along the fastest path through the physical wires of the network.

3. How Devices Find Each Other: IP Addresses and DNS

With billions of devices online, how does data sent from a server in California find its way directly to your phone? It relies on two main components:

IP Addresses (The Mailing Address)

Every single device connected to the internet is assigned a unique numerical label known as an IP (Internet Protocol) Address.

  • IPv4: The traditional format, written as four numbers separated by periods (e.g., 192.168.1.1).
  • IPv6: The newer format created because the world ran out of IPv4 addresses. It uses hexadecimal characters separated by colons (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334).

DNS (The Phonebook of the Internet)

Computers love numbers, but humans prefer names. You don't type 142.250.190.46 into your browser; you type google.com.

The Domain Name System (DNS) is a global directory that translates human-readable domain names (like google.com) into computer-readable IP addresses.

  1. You type example.com in your browser.
  2. Your computer asks a DNS Server: "What is the IP address for example.com?"
  3. The DNS Server looks it up and returns the IP address.
  4. Your browser uses that IP address to connect directly to the web server hosting the site.

4. How Data Travels: Packets and Protocols

When you download a file, watch a video, or view a web page, the data doesn't travel as one single massive block. If it did, a single hiccup in the connection would force you to restart the entire download.

Packet Switching

Instead, data is broken down into tiny chunks called packets. Each packet contains:

  • A piece of the actual content (payload).
  • A header containing the sender's IP address and the destination's IP address.
  • An order number (so your device knows how to assemble them back together).

These packets are sent across the internet. Because routers constantly calculate the fastest route, different packets of the same file might take completely different physical pathways across the globe, arriving out of order. Once all packets reach your device, they are reassembled into the seamless file or webpage you requested.

Protocols (The Rules of Communication)

For different devices to understand each other, they must follow standard rules called protocols:

Protocol Full Name Purpose
TCP/IP Transmission Control Protocol / Internet Protocol The fundamental standard of the internet. IP handles addressing and routing, while TCP ensures packets arrive safely and in order.
HTTP Hypertext Transfer Protocol Used by web browsers and servers to transmit web documents (HTML, images).
HTTPS Hypertext Transfer Protocol Secure An encrypted version of HTTP that protects sensitive data like passwords during transmission.

Summary: What happens when you search a website?

  1. The Request: You type a web address into your browser.
  2. The Lookup (DNS): Your device contacts a DNS server to translate that domain name into an IP address.
  3. The Connection (TCP/IP): Your browser uses that IP address to establish a secure connection with the web host server.
  4. The Split (Packets): The web server breaks the website's HTML, CSS, and image files into packets.
  5. The Journey (Routing): Routers across the globe guide these packets from network to network until they reach your local router.
  6. The Reassembly: Your browser receives the packets, reassembles them using TCP, and renders the complete webpage on your screen.
