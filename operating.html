Understanding Operating Systems

The bridge between human intention, applications, and physical hardware.

An Operating System (OS) is the core system software that acts as an intermediary between the computer hardware and the user (as well as their applications). Without an OS, you would have to write custom binary code just to get a computer to read a file from a disk or display a character on a screen.

Core Architecture of an OS

An operating system is typically split into distinct layers that separate user interactions from the physical hardware.

1. The Kernel

The Kernel is the heart of the operating system. It is the program that starts up first, remains in the main memory (RAM), and directly interacts with the physical hardware.

  • It manages memory allocation, schedules CPU tasks, and handles hardware communication through device drivers.
  • It runs in a privileged Kernel Mode to prevent user programs from directly modifying critical system resources and causing crashes.

2. The Shell (User Interface)

The Shell is the outer layer that wraps around the kernel. It takes input from the user and translates it into instructions the kernel can execute.

  • Command Line Interface (CLI): A text-based shell (e.g., Bash in Linux, Command Prompt/PowerShell in Windows) where users type commands.
  • Graphical User Interface (GUI): A visual shell (e.g., the Windows Desktop, macOS Finder) using windows, icons, and menus.

Principal Functions of an Operating System

To keep a computer running smoothly, the OS performs several critical resource management tasks behind the scenes:

Function What the OS Does
Process Management Allocates CPU time to active programs (processes). The OS uses schedulers to rapidly switch between tasks, giving the illusion that multiple programs are running simultaneously (multitasking).
Memory Management Allocates portions of the RAM to running programs and reclaims it when they close. If physical RAM runs out, the OS uses virtual memory (temporary space on the storage drive) to keep things running.
File System Management Organizes and tracks how data is stored on hard drives or SSDs. It translates raw blocks of physical storage into directories (folders) and files, managing access permissions and security.
Device Management Handles communication with external peripherals (keyboards, mice, printers, displays) via dedicated translators called device drivers.
Security & Isolation Ensures that one running application cannot accidentally or maliciously access the memory space or files of another application or user.

Types of Operating Systems

Depending on the device and its intended use, operating systems are designed differently:

  • Single-User, Single-Task: Designed for devices that only do one thing at a time (e.g., early MS-DOS, basic microcontrollers).
  • Multi-User, Multi-Tasking: Allows multiple users to log in and run separate applications simultaneously (e.g., Linux, Unix, Windows Server).
  • Real-Time Operating Systems (RTOS): Used where timing is critical and processing must happen with virtually zero latency (e.g., autopilot systems, industrial machinery, pacemaker controllers).
  • Mobile Operating Systems: Optimized for touchscreens, power conservation, and cellular connectivity (e.g., Android, iOS).

System Calls: When a user application (like a web browser or text editor) needs to write a file to the disk or connect to the internet, it cannot do so directly. Instead, it must make a System Call—a formal request asking the Kernel to perform the privileged operation on its behalf.
